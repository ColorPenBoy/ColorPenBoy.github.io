Git Tips：使用Git的一些基本技能

#git、ssh   

Tip0：本地已有文件了，如何加入git管理

git init

git add .

git commit -m "init xxx"

Tip1：查询历史commit log

使用 git log

这个commit-hash-id是每次commit的唯一标识码。

Tip2：版本回滚

如果某天产品突发奇想，让你加了一个十分难以实现的需求，代码散在整个项目到处都是。

就在你刚刚长舒一口气，终于完成了需求的时候，产品又过来了，笑眯眯的告诉你：这个需求砍掉了。。。掉了。。。了。。。

这个时候你已经忘记了，都在哪些文件中修改了代码，不可能一句句退回去了。是不是有想打死产品的冲动？这个时候，就需要用到版本回滚了

  • 先用 git log找到对应 commit 的 commit-hash-id（根据每次commit的注释找到，所以注释很重要，认真填写）

  • 然后用 git reset --hard <commit-hash-id> 回滚过去

轻轻松松滚回去，干干净净的项目。

Tip3：查询历史某天的某次 commit 都修改了什么

  • 老办法先用git log 找到对应天，对应 commit 的 commit-hash-id
  • 再用 git show <commit-hash-id>

就能找到，对应那次commit都修改了什么

Tip4：分支

  • git branch 查看本地分支
  • git branch -v 查看每个分支最后一次提交
  • git branch xxxxxx 新建分支
  • git checkout xxxxxx 切换到已有分支
  • git checkout -b xxxxxx 新建并切换到该分支，等同于上面两句命令

Tip5：合并分支

这里我们先设置一个目标：把dev分支代码合并到master分支上

  • 首先，将dev分支代码commit、push，都提交
  • 其次切换到master分支上 git checkout master
  • 然后将dev合并到master上 git merge dev
  • 合并成功后，切换回dev分支上

Tip5：给分支打Tag（标签）

  • 切换到需要打tag的分支上
  • git tag -a V2.0.0 -m "描述信息"
  • git push origin V2.0.0 此处 V2.0.0需要与上面语句中一致，否则无效
  • 切换到原分支

Tip6：删除分支

  • git branch -a查看所有分支（包含本地分支、远程分支）
  • 删除本地分支 git branch -D xxxxxx
  • 删除远程分支，首先 git branch -r -d origin/xxxxxx 然后 git push origin :xxxxxx (注意命令中的冒号“:”不要丢掉)
